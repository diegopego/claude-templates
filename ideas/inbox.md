# Inbox

The owner's scratchpad. Jot rough ideas here in any language, any quality — this is the one file in the repo exempt from the artifact-language rule.

An entry **graduates** when it becomes a proper spec in this directory (`Status: draft`); remove it from the inbox in the same change. The agent never reorganizes, rewrites, or deletes inbox entries on its own — graduation happens only when the owner asks.

---
- a skill update-manual é quem varre o repositório e atualiza os manuais e landing page. ela deve manter o último commit processado para poder trabalhar com o delta.
- o processo de update do site é iteração de demonstração da intenção em texto, depois na landing page e aguardar minha aprovação.
- os templates devem instruir que tarefas devem ser transformadas no esquema de spec-driven. se houver lacunas que considerar que deve perguntar, para poder criar uma spec ideal, pergunte imediatamente para o dono do projeto. 
- é preciso pensar em como fazer este setup. um projeto pode ter muitas variações. eu imagino que isto possa ser feito com um CLI, no estilo do próprio claude.
- tenho muitas escolhas enviesadas com nome de tecnologias, isso pode ser bom ou ruim. mas é fato que o desenvolvedor deve escolher durante o setup.
