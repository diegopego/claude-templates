# Inbox

The owner's scratchpad. Jot rough ideas here in any language, any quality — this is the one file in the repo exempt from the artifact-language rule.

An entry **graduates** when it becomes a proper spec in this directory (`Status: draft`); remove it from the inbox in the same change. The agent never reorganizes, rewrites, or deletes inbox entries on its own — graduation happens only when the owner asks.

- 2026-07-12 (friction, from the orderboard upgrade) — **`make upgrade` reads only the `**Source commit**:` line of the stamp.** Orderboard's stamp carried two commits: `Source commit: 29468ef` plus a prose line *"Reconciled up to 3e09fd2"*. The tool diffed from the older one and printed a superset — harmless, but the stamp had two answers and the machine read the wrong one. Either the installer learns the reconciliation line, or the stamp is required to keep `Source commit` as the single machine-readable truth and push everything else into a `## History` section (which is what the orderboard stamp now does, by hand). Note `template-version.md` is about to be redesigned anyway — `host-side-adoption.md` turns it into a disposition manifest, so fix it there rather than twice.
- 2026-07-12 (friction, from this repo's tooling) — **the pre-commit freshness hook fires on *any* `git commit` run through Bash**, including one inside an unrelated throwaway repo in `/tmp` used to exercise the installer. It blocked a `git init && git commit` that had nothing to do with these templates. The hook should scope itself to this repository (compare the repo root against `CLAUDE_PROJECT_DIR`, or bail when the cwd is outside it). Cost so far: a wasted test run and a confusing error; it will bite every time the installer is exercised end to end, which the charter *requires* before every `tools/` commit.
